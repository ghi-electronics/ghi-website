---
title: Display Drivers
---

Display-specific drivers in the **`GHIElectronics.TinyCLR.Drivers.*`** namespace extend TinyCLR's [graphics engine](../feature/graphics.mdx) and [low-level display support](../feature/displays.mdx) with ready-made drivers for common panels.

For devices with internal graphics support, see the [Graphics](../feature/graphics.mdx#virtual-displays) page — its virtual-displays example shows the full pattern for using the display drivers below. Just swap in the display controller you need.

For devices without internal native graphics, use [`BasicGraphics`](software-utility.mdx#basic-graphics) instead.

Displays that support **16BPP 5:6:5** (TinyCLR's native pixel format) get a `DrawBuffer` method. Displays with other formats (like 1BPP) get `DrawBufferNative` — convert the TinyCLR native format to whatever the display expects using the helpers documented in [Encoding & Decoding → Color space conversion](../feature/encoding-decoding.mdx#color-space-conversion).

:::tip
`BasicGraphics` supports 1BPP natively. For basic monochrome displays, it saves memory and skips the 16BPP → 1BPP conversion step.
:::

## ST7735
